# Презентация платформы «Наследники Победы»

## О платформе

**«Наследники Победы»** — это комплексная веб-платформа для проведения регионального творческого конкурса среди школьников Севастополя, организованного Севастопольским отделением «Боевого Братства».

Платформа разработана с нуля и представляет собой полнофункциональное решение для управления конкурсными работами, включающее в себя все необходимые компоненты современного веб-приложения:

### Ключевые возможности платформы

**Полный цикл работы с конкурсными материалами:**
- Регистрация и авторизация участников (школьников) с хранением всех необходимых данных
- Подача работ с загрузкой файлов различных форматов (сочинения и рисунки)
- Автоматическое назначение работ экспертам или ручное распределение
- Система оценки работ с возможностью оставлять комментарии
- Выгрузка всех данных в Excel для отчётности и архивации

**Разграничение прав доступа:**
- **Администратор** — полный контроль над системой: управление пользователями, назначение экспертов, просмотр статистики, экспорт данных
- **Эксперт** — просмотр назначенных работ, выставление оценок и комментариев
- **Школьник (участник)** — регистрация, подача работ, отслеживание статуса и результатов оценки

**Email-уведомления:**
- При регистрации участника
- При подаче работы
- При назначении работы эксперту
- При выставлении оценки
- При сбросе пароля

**Удобство управления:**
- Интуитивно понятный интерфейс в патриотическом стиле
- Адаптивная вёрстка для работы с любых устройств
- Графики и статистика в реальном времени
- Система фильтрации и поиска
- Централизованное управление настройками (дедлайны, шкала оценок)

**Техническая готовность:**
Проект полностью готов к развёртыванию на production-сервере. В настоящий момент платформа работает в локальной среде разработки, что позволяет оперативно вносить правки и дополнения по результатам тестирования и обратной связи от заказчика.

## Обзор интерфейса платформы

### Главная страница

Пользователей встречает современная целевая страница (лендинг) с информацией о конкурсе. Здесь размещены логотипы организаторов — города Севастополь и «Боевого Братства», название конкурса «Наследники Победы» и актуальная информация о дедлайне подачи работ.

На главной странице представлены две номинации конкурса: «Великая Отечественная война» и «Специальная военная операция». Каждая номинация имеет своё краткое описание и визуальное оформление в соответствующей стилистике.

Для участников доступна кнопка «Принять участие», которая перенаправляет на регистрацию или в личный кабинет (если пользователь уже авторизован). В шапке сайта размещена кнопка «Войти» для авторизованных пользователей.

Также на странице представлена пошаговая инструкция участия в конкурсе: регистрация → подача работы → оценка экспертами → объявление результатов.

### Страница входа

Форма авторизации позволяет пользователям войти в систему, указав email и пароль. Реализована защита от brute-force атак через систему rate limiting (не более 10 попыток входа в минуту). При неверных данных система выводит соответствующее сообщение об ошибке.

Присутствует ссылка «Забыли пароль?» для восстановления доступа и кнопка «Зарегистрироваться» для новых участников.

### Страница регистрации

Форма регистрации собирает всю необходимую информацию об участнике:
- ФИО (полное имя)
- Email для входа и уведомлений
- Телефон
- Название школы
- Класс обучения
- Пароль (с подтверждением)

Обязательным условием является принятие политики конфиденциальности — без этого регистрация невозможна. После успешной регистрации на указанный email отправляется приветственное письмо, а участник автоматически перенаправляется в свой личный кабинет.

Система также защищена от множественных попыток регистрации (не более 5 попыток в минуту).

### Страница восстановления пароля

Если участник забыл пароль, он может восстановить доступ через форму сброса пароля. Достаточно указать email, использованный при регистрации. Система отправит на почту ссылку для создания нового пароля. Ссылка действительна ограниченное время и может быть использована только один раз.

Этот функционал также защищён rate limiting (не более 3 попыток в минуту) для предотвращения спама.

### Политика конфиденциальности

Полный текст политики конфиденциальности доступен для ознакомления. Документ описывает принципы обработки персональных данных, права участников и обязательства организаторов в соответствии с законодательством РФ.

## Панель администратора

### Главная страница (дашборд)

Центральный экран администратора содержит исчерпывающую статистику о состоянии конкурса:

**Основные показатели:**
- Количество зарегистрированных участников
- Количество активных экспертов
- Общее число поданных работ
- Средний балл по всем оценённым работам

**Визуализация данных:**
- Круговая диаграмма распределения работ по номинациям (ВОВ / СВО)
- Столбчатая диаграмма работ по статусам (На модерации, На проверке, Оценено)
- Горизонтальная диаграмма топ-10 школ по количеству поданных работ

**Быстрые действия:**
Набор кнопок для оперативного перехода к основным разделам: Пользователи, Работы, Создание эксперта, Настройки.

Если в системе есть работы без назначенного эксперта, выводится предупреждение с кнопкой автораспределения.

В шапке страницы доступны кнопки экспорта данных в Excel: «Экспорт работ» (все работы) и «Экспорт результатов» (только оценённые работы).

### Управление пользователями

Страница содержит полный список всех зарегистрированных пользователей системы с возможностью фильтрации по ролям:
- Все пользователи
- Только участники (школьники)
- Только эксперты
- Только администраторы

Для каждого пользователя отображается:
- ФИО
- Email
- Роль в системе
- Школа и класс (для участников)
- Статус (активен / заблокирован)

**Доступные действия:**
- Блокировка/разблокировка пользователя
- Сброс пароля с отправкой ссылки на email
- Поиск по имени или email

Заблокированный пользователь не может войти в систему до снятия блокировки.

### Управление работами

Центральная страница для работы с конкурсными материалами. Реализована система фильтрации:
- Все работы
- Без эксперта (ожидают назначения)
- На проверке (назначены экспертам, но не оценены)
- Оценено (имеют финальную оценку)

**Информация о каждой работе:**
- Название
- ФИО автора, школа, класс
- Номинация (ВОВ / СВО)
- Тип работы (Сочинение / Рисунок)
- Статус
- Назначенный эксперт
- Выставленная оценка (если есть)

**Действия администратора:**
- Скачать файл работы для проверки
- Назначить эксперта вручную (выбор из выпадающего списка + массовое назначение через чекбоксы)
- Снять эксперта с работы (только если работа ещё не оценена)
- Автораспределение — алгоритм равномерно распределяет все неназначенные работы между активными экспертами с учётом их текущей нагрузки

При назначении работы эксперту на его email автоматически отправляется уведомление.

### Создание эксперта

Форма для добавления нового эксперта в систему. Администратор заполняет:
- ФИО эксперта
- Email (будет использоваться для входа)
- Телефон (опционально)
- Пароль — можно ввести вручную или сгенерировать случайный

После создания администратор должен передать эксперту его учётные данные (email и пароль) для входа в систему.

### Настройки системы

Централизованное управление ключевыми параметрами конкурса:

**Дедлайн подачи работ:**
Установка даты и времени окончания приёма работ. После наступления дедлайна участники не смогут подавать новые работы или редактировать существующие.

**Максимальный балл:**
Настройка шкалы оценивания (например, от 1 до 10). Эксперты при оценке работ смогут выставлять баллы только в рамках установленной шкалы.

Все изменения сохраняются мгновенно и отражаются на всех страницах системы.

## Личный кабинет участника (школьника)

### Главная страница кабинета

После входа в систему участник попадает в свой личный кабинет. В шапке страницы отображается его ФИО, а также информация о школе и классе обучения.

Главная кнопка — **«Подать работу»** — позволяет перейти к форме подачи конкурсной работы. Эта кнопка доступна до наступления дедлайна, установленного администратором.

**Раздел «Мои работы»:**
Здесь отображаются все работы, поданные участником на конкурс. Для каждой работы видна полная информация:

- Название работы
- Номинация (Великая Отечественная война / Специальная военная операция)
- Тип работы (Сочинение / Рисунок)
- Имя файла и его размер
- Дата подачи
- Текущий статус работы:
  - **На проверке** (жёлтый) — работа назначена эксперту, ожидает оценки
  - **Оценено** (зелёный) — работа получила финальную оценку

**Для оценённых работ** отображается:
- Выставленная оценка (крупно, золотым цветом)
- Количество баллов из максимально возможных

**Доступные действия:**
- **Скачать** — загрузить файл своей работы для проверки
- **Редактирование** — пока работа находится в статусе «На проверке», участник может изменить название или заменить файл работы (кнопка «Редактировать» появляется при наведении)

Если работа уже оценена, редактирование недоступно — отображается только оценка и кнопка скачивания.

### Форма подачи работы

При нажатии кнопки «Подать работу» открывается специальная форма с пошаговым выбором:

**1. Название работы**  
Текстовое поле для ввода названия (например, «Письмо прадедушке»).

**2. Выбор номинации**  
Два варианта в виде крупных кнопок:
- **ВОВ** — Великая Отечественная война (с красной звездой)
- **СВО** — Специальная военная операция (с золотой звездой)

**3. Тип работы**  
Два варианта:
- **Сочинение** — текстовая работа (с иконкой документа)
- **Рисунок** — художественная работа (с иконкой палитры)

**4. Загрузка файла**  
Область drag & drop с возможностью:
- Перетащить файл мышкой
- Нажать для выбора файла через диалог системы

Поддерживаемые форматы:
- Для сочинений: PDF, DOC, DOCX
- Для рисунков: JPEG, PNG, GIF
- Максимальный размер: 15 МБ

После загрузки отображается имя файла и его размер.

**Кнопка «Подать работу»**  
Крупная золотая кнопка внизу формы. После нажатия:
- Работа сохраняется в системе
- Файл загружается в защищённое хранилище
- Участнику на email отправляется подтверждение о принятии работы
- Работа появляется в разделе «Мои работы» со статусом «На проверке»

**Ограничения:**
- Участник может подать **только одну работу на каждую номинацию** (максимум 2 работы: одну на ВОВ, одну на СВО)
- После дедлайна подача новых работ блокируется
- Редактирование возможно только до выставления оценки экспертом

## Личный кабинет эксперта

### Список назначенных работ

После входа эксперт попадает в свой кабинет, где видит:

**Статистика работ:**
- Назначено всего работ
- Ожидает оценки
- Уже оценено

**Список работ:**
Для каждой работы отображается:
- Название
- ФИО автора
- Школа и класс
- Номинация и тип работы

**Действия:**
- «Скачать» — загрузка файла работы для ознакомления
- «Оценить» — открытие формы оценки

Эксперт видит только те работы, которые назначены ему администратором.

### Окно оценки работы

При нажатии кнопки «Оценить» открывается модальное окно с формой:

**Название работы** — для контекста

**Слайдер оценки** — эксперт перемещает ползунок, выставляя оценку от минимума до максимума (например, от 1 до 10). Текущее значение отображается крупно над слайдером.

**Комментарий (необязательно)** — текстовое поле для развёрнутого отзыва или обоснования оценки.

**Кнопки:**
- «Отмена» — закрыть окно без сохранения
- «Сохранить» — сохранить оценку и комментарий

После сохранения оценки:
- Работа получает статус «Оценено»
- Участнику на email отправляется уведомление с результатами
- Работа исчезает из списка «Ожидает оценки» и переходит в «Оценено»

## Email-уведомления

Система автоматически отправляет email-уведомления на всех этапах работы:

### Для участников:
- **При регистрации** — приветственное письмо с подтверждением регистрации
- **При подаче работы** — уведомление о том, что работа принята и отправлена на проверку
- **При оценке работы** — письмо с результатами оценки, комментарием эксперта и ссылкой в личный кабинет
- **При сбросе пароля** — ссылка для создания нового пароля

### Для экспертов:
- **При назначении работы** — уведомление о том, что им назначена новая работа для оценки

Все письма оформлены в едином стиле и содержат логотипы организаторов. В локальной версии письма можно просмотреть через веб-интерфейс MailHog, где видны все отправленные сообщения с полным содержимым и заголовками.

## Техническая реализация

Платформа построена на современном технологическом стеке:

**Frontend:**
- React.js + Vite — быстрая разработка и оптимальная производительность
- Tailwind CSS — современный дизайн
- TypeScript — типобезопасность кода

**Backend:**
- NestJS — надёжный фреймворк для enterprise-решений
- PostgreSQL — промышленная СУБД
- Prisma ORM — удобная работа с базой данных
- JWT-авторизация с refresh-токенами

**Инфраструктура:**
- Docker Compose для локальной разработки
- MinIO (S3-совместимое хранилище) для файлов работ
- MailHog для тестирования email-уведомлений

**Безопасность:**
- Rate Limiting для защиты от brute-force атак
- Хеширование паролей (bcrypt)
- Защита API через JWT-токены
- Разграничение доступа по ролям

**Готовность к production:**
Проект имеет полную документацию, включая:
- Руководство пользователя
- Чек-лист приёмки (89 пунктов проверки)
- Архитектурные диаграммы
- ER-диаграмму базы данных
- Сценарий видео-демонстрации

Все компоненты протестированы:
- Unit-тесты для backend (Jest)
- End-to-End тесты для frontend (Playwright)

Проект готов к развёртыванию на production-сервере. Текущая локальная версия используется для финального тестирования и внесения возможных правок перед публикацией.
